Quick Start

Running in Under 10 Minutes

Full stack — database, API, worker, and frontend — with one terminal each. You need Docker, Node 20, and Python 3.11.

1

Clone and configure

Copy the env template and drop in your Claude API key. Get one free at console.anthropic.com.

bash
# Clone the repo
$git clone https://github.com/thebardchat/Greenfield.git && cd Greenfield
$cp .env.example .env
# Open .env and set these two values:
ANTHROPIC_API_KEY=sk-ant-api03-...
JWT_SECRET=some-long-random-string-here
2

Start infrastructure

Docker Compose spins up PostgreSQL (with pgvector), Redis, and MinIO. Wait ~10 seconds for Postgres to initialize.

bash
$docker-compose up -d postgres redis minio
✔ Container greenfield-postgres-1 Started
✔ Container greenfield-redis-1 Started
✔ Container greenfield-minio-1 Started
3

Run migrations + seed

14 SQL migrations build the full schema including pgvector embeddings table. The seed loads demo orgs, users, and claims.

bash
# Run all migrations
$for f in db/migrations/*.sql; do docker-compose exec -T postgres psql -U claimcruncher -d claimcruncher < "$f"; done
CREATE TABLE ... CREATE INDEX ... (×14)
$python db/seeds/hash_passwords.py && docker-compose exec -T postgres psql -U claimcruncher -d claimcruncher < db/seeds/seed.sql
✓ Demo data seeded — login: admin@claimcruncher.com / Admin1234!
4

Start the API, worker, and frontend

Three terminals. Once all three are running, Claim Cruncher is live.

Terminal 1 — API
$cd apps/api && pip install -e ".[dev]"
$uvicorn app.main:app --reload --port 8000
INFO: Uvicorn running on http://0.0.0.0:8000
Terminal 2 — OCR Worker
$cd apps/worker && pip install -e ".[dev]"
$arq app.main.WorkerSettings
Worker started. Redis: redis://localhost:6380/0
Terminal 3 — Frontend
$cd apps/web && npm install && npm run dev
➜ Local: http://localhost:5173/
5

Verify Cruncher AI is connected

One curl to confirm Claude is wired up and streaming is ready.

bash
$curl -s http://localhost:8000/api/cruncher/health | python -m json.tool
{
  "status": "ok",
  "cruncher_enabled": true,
  "model": "claude-sonnet-4-6",
  "flag_model": "claude-haiku-4-5-20251001",
  "rag_enabled": true
}
✓ Open http://localhost:5173 → log in → click any claim → Cruncher AI
API Reference

REST Endpoints

Full OpenAPI docs at http://localhost:8000/docs when running locally.

MethodPathDescription
POST/api/auth/loginLogin, returns JWT access + refresh tokens
GET/api/claimsList claims (org-scoped, paginated, filterable)
POST/api/claimsCreate new claim with patient + payer info
GET/api/claims/{id}Get full claim with lines, documents, tickets
POST/api/documents/uploadUpload PDF/image → SHA-256 dedup → OCR queue
GET/api/documents/{id}/ocrGet OCR status, extracted text, structured JSON
POST/api/cruncher/chatStreaming SSE — Claude tool-use agentic chat ✦ AI

POST/api/cruncher/denial-analysis/{id}Denial reason analysis + appeal letter language ✦ AI
POST/api/cruncher/parse-eobParse EOB document text → structured JSON ✦ AI
GET/api/reports/claims-summaryStatus breakdown, collection rate, denial rate
GET/api/reports/denial-trendsTop denial reasons + appeal success rates

GET/api/cruncher/healthCheck Claude connection, models, RAG status
